# 유저 시나리오

> 우아한테크코스 클로드 코드 원정대 — 나무위키 스타일 지식 베이스 서비스

## 페이지 구성

| 페이지 | 경로 | 설명 |
|--------|------|------|
| 메인 (랜딩) | `/` | 서비스 소개 + GNB |
| Docs | `/docs` | 사이드바 트리 + 문서 목록 |
| 문서 상세 | `/docs/:slug` | 마크다운 렌더링 + TOC + 코드 하이라이팅 |
| Blog | `/blog` | 멤버 블로그 카드 목록 |
| 블로그 상세 | `/blog/:slug` | 블로그 포스트 상세 |
| 어드민 로그인 | `/admin` | 어드민 전용 진입점 (별도 URL) |
| 어드민 대시보드 | `/admin/dashboard` | 문서 관리 |
| 어드민 에디터 | `/admin/editor/:slug?` | 마크다운 에디터 (신규 작성 / 수정) |

---

## 시나리오 1: 방문자 — 문서 탐색

> 비로그인 사용자가 메인 페이지에서 GNB를 통해 문서를 탐색하고 읽는 흐름

### 흐름

```
메인 페이지 (랜딩)
  ├─ [GNB] → Docs 페이지
  │            ├─ 사이드바 트리에서 카테고리 탐색
  │            └─ 문서 클릭 → 문서 상세 (Details)
  │                             ├─ TOC로 섹션 이동
  │                             └─ URL 공유 (slug 기반 permalink)
  │
  └─ [GNB] → Blog 페이지
               └─ 카드 클릭 → 블로그 상세
```

### 상세 단계

1. 사용자가 `/` (메인 페이지)에 진입한다.
2. 메인 페이지에는 서비스 소개 글과 GNB만 존재한다.
3. GNB에서 **Docs** 또는 **Blog**를 클릭하여 이동한다.
4. **Docs 경로**: 사이드바 트리 네비게이션으로 계층적 문서 구조를 탐색한다.
5. 문서를 클릭하면 `/docs/:slug` 형태의 상세 페이지로 이동한다.
6. 상세 페이지에서 TOC(헤딩 기반 목차)를 통해 원하는 섹션으로 빠르게 이동한다.
7. 코드 블록은 syntax highlighting이 적용된 상태로 렌더링된다.
8. slug 기반 permalink를 복사하여 다른 사람에게 공유할 수 있다.

### 관련 기능

- 사이드바 트리 네비게이션 (계층적 문서 구조)
- 문서 내 목차 (TOC, 헤딩 기반)
- 코드 구문 강조 (syntax highlighting)
- 고유 URL 체계 (slug 기반 permalink)

## 시나리오 2: 어드민 — 문서 작성 및 수정

> 어드민이 별도 URL로 진입하여 인증 후 마크다운 에디터로 문서를 관리하는 흐름

### 흐름

```
/admin URL 직접 진입
  └─ 로그인 (인증)
       └─ 어드민 대시보드
            ├─ 새 문서 작성
            │    └─ 마크다운 에디터
            │         └─ 미리보기 + 메타 설정 (slug, 카테고리, 태그)
            │              └─ 저장 / 발행
            │
            └─ 기존 문서 수정
                 └─ 마크다운 에디터 (기존 내용 로드)
                      └─ 미리보기 + 메타 설정
                           └─ 저장 (편집 기록 자동 생성)
```

### 상세 단계

1. 어드민이 `/admin` URL로 직접 진입한다 (메인 페이지와 별개 경로).
2. 로그인 페이지에서 인증을 수행한다.
3. 인증 성공 시 어드민 대시보드로 이동한다.
4. **새 문서 작성**: 빈 마크다운 에디터가 열린다.
5. **기존 문서 수정**: 문서 목록에서 선택하면 기존 내용이 에디터에 로드된다.
6. 마크다운 에디터(외부 라이브러리)에서 내용을 작성한다.
7. 미리보기로 렌더링 결과를 확인한다.
8. 메타 정보(slug, 카테고리, 태그)를 설정한다.
9. 저장 또는 발행 버튼을 클릭한다.
10. 수정의 경우 편집 기록이 서버에 자동으로 생성된다.

### 관련 기능

- 어드민 인증 (별도 URL 진입)
- 마크다운 에디터 (외부 라이브러리)
- CRUD API (문서 생성, 수정, 삭제)
- 편집 기록 (서버사이드)

## 시나리오 3: 검색 — 문서 내 텍스트 검색

> 사용자가 Docs 또는 Blog 페이지의 GNB 검색을 통해 원하는 문서를 찾는 흐름

### 흐름

```
Docs / Blog 페이지
  └─ [GNB] 검색 아이콘 클릭
       └─ 검색어 입력 (서버사이드 텍스트 검색)
            └─ 검색 결과 목록 (제목 + 본문 하이라이트)
                 ├─ 결과 클릭 → 문서 상세로 이동
                 └─ 결과 없음 → 검색어 수정 후 재검색
```

### 상세 단계

1. 사용자가 Docs 또는 Blog 페이지에서 GNB의 검색 아이콘을 클릭한다.
2. 검색 입력창에 키워드를 입력한다.
3. 서버사이드 텍스트 검색 API가 호출된다.
4. 검색 결과가 목록으로 표시된다 (제목 + 본문 하이라이트 매칭).
5. 원하는 결과를 클릭하면 해당 문서 상세 페이지로 이동한다.
6. 결과가 없는 경우 검색어를 수정하여 재검색한다.

### 관련 기능

- 문서 내 텍스트 검색
- 서버사이드 검색 API
- 검색 결과 하이라이트

---

## 비기능 요구사항

- 모든 문서 페이지는 slug 기반 permalink를 가진다 (예: `/docs/claude-code-tips`).
- 메인 페이지에는 검색 기능이 없으며, 서비스 소개와 GNB만 존재한다.
- 어드민 페이지는 일반 사용자에게 노출되지 않으며, 별도 URL로만 접근 가능하다.
- 편집 기록은 서버에 저장되지만, 문서 상세 페이지에서 사용자에게 노출하지 않는다.
