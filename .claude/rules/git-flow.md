# Git & 커밋 규칙

GitHub 워크플로우, 커밋 메시지, 주석 관련 전체 규칙.

---

## 1. GitHub 전략

### PR 설정
- **Reviewers**: 기본적으로 '모두'로 설정 (상세 내용은 코드 리뷰 가이드 참고)
- **Assignees**: PR 작성자가 본인에게 할당 (공동 작업 시 추가 담당자 지정)
- **Labels**: PR에 적절한 라벨을 반드시 추가하여 작업 유형 명확화

### 자동화
- PR Description 템플릿과 라벨링을 자동화

## 2. 커밋 메시지 컨벤션

### 형식
```
<type>(<scope>): <작업 내용>
```

예시:
```
feat(admin): 어드민 로그인 로직 구현
fix(auth): 토큰 만료 시 재발급 실패 버그 수정
refactor(user): UserService 의존성 주입 구조로 변경
```

### Type 종류

| Type       | 용도                                                 |
| ---------- | -------------------------------------------------- |
| `feat`     | 기능 구현 또는 새로운 기능 추가                          |
| `fix`      | 버그 및 오류 수정                                     |
| `docs`     | README.md, 주석 작성 등 문서 관련 작업                 |
| `style`    | 린트, 코드 포맷팅 관련 (로직 변경 없음)                |
| `refactor` | 코드 리팩토링 (기능 변화 없이 코드 개선)                |
| `setting`  | 빌드 수행, 패키지 설치, 환경 설정 수정 등              |
| `chore`    | 기타 작업                                            |

### 자동 검증
이 컨벤션은 `.claude/hooks/commit-msg-validate.sh`에서 자동 검증된다.
Claude가 `git commit -m "..."` 실행 시 형식에 맞지 않으면 차단된다.

## 3. 주석 컨벤션

### 기본 원칙
- 복잡한 로직의 클래스·함수는 **JSDoc**으로 명확히 작성
- 코드가 설명하지 못하는 **의도**, 제약 조건, 우회 사유 등 *왜*를 담는다

### AI 주석 금지
- JSDoc 외의 주석을 AI로 자동 생성하지 않는다
- 필요한 경우 개발자가 직접 맥락을 담아 작성
