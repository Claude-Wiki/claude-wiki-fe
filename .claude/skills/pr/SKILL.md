# pr 작성 커맨트

이 커맨트는 HEAD의 커밋과 develop 브랜치의 커밋의 변경사항을 파악하여 pr 내용을 작성하고, pr을 생성하는 커맨트입니다.

feature 브랜치에서 develop 브랜치로 pr을 생성하는 것을 기준으로 작성되어 있습니다.

## 이 커맨트가 하는 일

1. **변경사항 확인** - 코드의 변경사항을 명확히 확인

- `git diff develop..HEAD` 명령어를 호출하여 변경된 코드 확인
- `git log develop..feature` 명령어를 호출하여 쌓인 커밋 메시지 확인

2. **적절한 pr 형식에 맞추어 pr 문서 작성 및 PR 생성** - 확인된 변경사항 및 커밋 내용을 전체적으로 확인하여 pr 문서 작성

- 작성된 pr 문서를 임시 파일(예: `.github/PR_BODY.md`)로 저장합니다.
- `gh pr create` 명령어를 사용하여 아래 옵션으로 PR을 생성합니다.
  - `--base dev` : dev 브랜치를 대상으로 설정
  - `--head <현재 브랜치>` : 현재 feature 브랜치에서 PR 생성
  - `--body-file .github/PR_BODY.md` : 작성한 PR 문서 사용
  - `--assignee @me` : 본인을 assignee로 지정
  - `--reviewer <팀원1>,<팀원2>,...` : 팀원 전체를 리뷰어로 지정
  - `--label <라벨명>` : 변경사항에 맞는 적절한 라벨 지정

## 작동 방식

- 변경사항을 명확하고 정확하게 파악합니다.
  - `git diff develop..HEAD` 명령어를 호출하여 변경된 코드 확인
  - `git log develop..feature` 명령어를 호출하여 쌓인 커밋 메시지 확인

- 확인된 변경사항 및 커밋 내용을 전체적으로 확인하여 pr문서를 작성하여 pr을 생성합니다.
  - 해당 파일이 존재하지 않는경우 새롭게 만듧니다.
  - 해당 파일이 이미 존재하는 경우 내용을 덮어씁니다.

## pr 문서 형식

```
- #[이슈번호]

*주요 변경사항 및 pr 내용 요약(초록)내용 작성*
한 문장에서 두 문장으로 작성. **단 높임말 사용.**

## 주요 변경사항

### [변경사항 1]
만약 변경 사항이 많다면 개괄식이 아닌 h3로 하나씩 나누어서 줄글로 설명
설명시에는 ~하였습니다. 방식의 **높임말 사용**

## 리뷰 주의사항

- 리뷰어에게 중점적으로 봐주길 원하는 부분 개괄적으로 작성
- 필요 없다면 생략 가능
- **단 높임말 사용.**

close #[이슈번호]
```

## pr 문서 작성 예시

```md
- #13

## 주요 변경 사항

프로필 사진 업로드 및 제거 기능을 구현하였습니다.

### 프로필 사진 업로드

프로필 사진의 경우 s3를 통하여 관리하도록 하였습니다.
이에 따라서 presignedUrl을 받은 이후, 프론트에서 파일을 업로드 하고 올바르게 업로드가 되었다는 메시지를 백엔드에게 보내는 방식입니다.
여기에서 업로드라는 동작 하나에 3개의 api 호출을 해야합니다.
현재로서는 3개의 api를 하나의 비동기 함수로 만들고, 이를 핸들러에서 관리하도록 구현하였습니다.
다만 이는 기존의 api(axios) -> query hook -> components 의 계층을 이루던 컨밴션에 위배되는 코드이므로, 추후 개선하도록 하겠습니다.

### 업로드할 프로필 사진 수정

요구사항에 따라서 업로드할 프로필의 크기 및 위치를 수정할 수 있어야 했습니다.
이에 따라서 GestureDetector와 Animated.Image를 통하여 이미지를 이동할 수 있도록 하였습니다.
이후, 이동한 위치 및 확대한 크기를 계산하고, image-manipulator를 사용하여 해당 위치에 해당 크기만큼 crop하도록 하였습니다.

## 리뷰 주의사항

- 프로필 사진 업로드 및 제거 기능이 요구사항에 맞게 구현되었는지 확인 부탁드립니다.
- 프로필 사진 업로드의 경우 3개의 api 호출이 필요한데, 이를 하나의 비동기 함수로 구현하였는데, 이 부분이 컨밴션에 위배되는 코드이므로, 추후 개선할 예정입니다. 이에 대해서도 리뷰 부탁드립니다.

close #13
```

## 주의사항

- **[매우 중요]** 해당 스킬은 절대로 코드를 작성하거나 수정하지 않습니다. 
- **[매우 중요]** 만약 해당 스킬을 feature 브랜치가 아닌 다른 브랜치에서 실행할 경우, 작업 시작 전 사용자에게 경고 메시지를 출력하고 종료합니다.
